# КПФ Дашборд — Старый Хинкалич

Аналитический дашборд для сети ресторанов. Данные из iiko API (OLAP-отчёты).

## Быстрый старт

### 1. Установи Node.js (если нет)

Скачай с https://nodejs.org — нужна версия **18+**.

Проверь:
```bash
node -v    # должно быть v18.x или выше
npm -v     # должно быть 9.x или выше
```

### 2. Установи зависимости

```bash
cd kpf-dashboard
npm install
```

### 3. Запусти

```bash
npm run dev
```

Откроется http://localhost:3000 — дашборд готов.

Логин: любой логин + пароль (сейчас mock-авторизация).

### 4. Сборка для продакшена

```bash
npm run build
```

Готовые файлы будут в папке `dist/` — можно хостить на любом статическом сервере.

## Структура проекта

```
kpf-dashboard/
├── index.html          # HTML точка входа
├── package.json        # Зависимости
├── tsconfig.json       # TypeScript конфиг
├── vite.config.ts      # Vite конфиг (dev-сервер)
└── src/
    ├── main.tsx        # React entry point
    ├── App.tsx         # Весь дашборд (компоненты + логика)
    └── vite-env.d.ts   # TypeScript типы для Vite
```

## Что внутри

| Раздел | Описание |
|--------|----------|
| **Сводка** | Все рестораны: выручка, план, LC%, foodcost, хинкали |
| **По ресторану** | Детальный КПФ по дням + графики |
| **ФОТ / LC** | Разбивка по категориям, донат, роли |
| **Внимание** | Алерты + сотрудники без тарифа |

## Следующие шаги: подключение к iiko

Сейчас работает на mock-данных. Для реальных данных нужен бэкенд-прокси
(FastAPI / Express), который делает OLAP-запросы к iiko и отдаёт JSON.
